MVC Architecture

 
תבנית Model-View-Controller (בקיצור MVC) היא תבנית עיצוב המשמשת להפשטת אפליקציה כלשהי חלוקת היישום מחולקת לשלושה חלקים, מודל, תצוגה ובקר. בשיטה זו התלות בין הממשק משתמש לשאר חלקי התוכנה פוחתת, וכך מצומצמת התלות בין המרכיבים, התחזוקה פוחתת ניתן לעשות שימוש חוזר בחלקים השונים.
היתרון הגדול והמרכזי ביותר של MVC הוא כמובן הפרדת השכבות והיכולת לנתק לחלוטין את אופן עיבוד המידע (שכבת ה control) משכבת ההצגה שלו למשתמש (view).
ההפרדה בין השכבות נותנת יכולת לרכז את הפיתוח לדוג' בשכבת ה VIEW בלי תלות ב BUSINESS LOGIC.
בפיתוח תחת מודל ה MVC כל שמפתח הלוגיקה נדרש לדעת הוא מהם הארגומנטים אותם הוא מקבל משכבת ה view ואיזה מידע מועבר בתוכם. controller עצמו אינו נדרש עוד לזהות בעצמו את האובייקט שהפעיל אותו, הוא אינו נדרש לדוגמא לנתח איזה כפתור נלחץ, הוא רק נדרש לקבל את הקלט שהוא מצפה לו, להפעיל את העיבוד ולהחזיר לשכבת ה VIEW את הפלט המתאים

בין יתרונות ה MVC  ניתן למנות  -   

  1. תהליך פיתוח אפליקציות WEB מהיר יותר.
  2. תומך בטכניקת פעילות רקע אסינכרונית.
  3. אידיאלי לSCALE  - פיתוח אפליקציות WEB גדולות.
  4. החזרת הDATA ללא צורך בפירמוט.
  5. שינוי בחלק קטן לא משפיע על כל המודל.
  6. פיתוח בו זמנית - הפרדת חלקי האפליקציה לפי טכנולוגיות וחלוקת הפיתוח יעילה צד ה VIEW ינתן למפתחי פרונט וצד CONTROL ינתן למפתחי צד שרת.
  7. יעילות וקלות בביצוע UNIT TESTING.
  8. לכידות גבוהה - מאפשר איחוד לוגי של פעולות דומות ב CONTROLLER, כמו כן,  VIEWS שונים של המודל מקובצים יחד.

חסרונות - 

  1. ניווט קוד - ניווט המסגרת יכול להיות מורכב מכיוון שהוא מציג שכבות הפשטה חדשות ומחייב את המשתמשים להסתגל לקריטריוני הפירוק של MVC.
  2. ייצוגים מרובים - פירוק תכונה לשלושה חפצים גורמת לפיזור. לפיכך, דרישה למפתחים לשמור על עקביות של מספר ייצוגים בבת אחת.

 difference (in Javascript) between "==" & "===" logical operators

"=="  -  משמש להשוואה בין שני משתנים ללא קשר לסוג הנתונים של המשתנה.

"===" -  משמש להשוואה בין שני משתנים אך הדבר יבדוק סוג קפדני - STRICT TYPE, מה שאומר שהוא יבדוק את סוג הנתונים וישווה שני ערכים.

דוגמה: 

var number = 100;

if (number == 100).....

כלומר במצב שמשווים "100" = 100  לעומת   100=100     - זה יהיה אותה תוצאה.

לעומת זאת כאשר משתמשים באופרטור "==="

אזי יבדק גם סוג הנתון  -   

במצב שמשווים "100" = 100    (לא שווים) .  לעומת   100=100 (שווים)     - זה  לא יהיה אותה תוצאה.



Extend the "Array" class in Javascript
with a new method that calculates and returns the sum of the Array's items


      class TravellerCollection extends Array {
        sum(key) {
            return this.reduce((a, b) => a + (b[key] || 0), 0);
        }
    }
    const traveler = new TravellerCollection(...[
        {  description: 'Senior', Amount: 50},
        {  description: 'Senior', Amount: 50},
        {  description: 'Adult', Amount: 75},
        {  description: 'Child', Amount: 35},
        {  description: 'Infant', Amount: 25 }, 
    ]);
    
    console.log(traveler.sum('Amount'));
